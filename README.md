# Redbus Website -- Manual Testing Project

This project focuses on end‑to‑end manual testing of the Redbus website
booking workflow. The objective is to validate functionality, usability
and user‑journey stability across critical flows such as:

-   Searching buses
-   Applying filters & sorting
-   Viewing seat layouts & selecting seats
-   Filling passenger details
-   Proceeding to payment gateway
-   Handling invalid input & edge cases
-   Behaviour during network issues
-   UI & UX review

## Deliverables Included

-   Test Plan
-   Test Cases (Positive, Negative & Edge Scenarios)
-   Bug / Observation Report
-   Test Summary Report

## Tools Used

-   Google Chrome (Desktop & Mobile)
-   Excel 
-   PDF Documentation

## Project Scope

-   Homepage Search & Validation
-   Search Results + Filters
-   Seat Layout & Fare Calculation
-   Passenger Details Form
-   Payment Redirect Behaviour
-   Modify Search
-   Network Handling
-   Usability & UI Review
